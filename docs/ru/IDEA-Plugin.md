Интеграция с IntelliJ IDEA (Plugin Support)
===========================================

Плагин **Integration Testing** превращает IntelliJ IDEA в интерактивную среду для разработки и записи тестов. Он
автоматически внедряет JVM-аргументы во все запуски **JUnit** и **TestNG** , позволяя управлять поведением тестов
визуально, без правки конфигурационных файлов.

## 1. Окно инструментов (Tool Window)

После установки плагина в IDEA появляется вкладка **Integration Testing** . Все настройки применяются к следующему
запуску любого интеграционного теста в проекте.

### Режимы сравнения (Assertion)

Определяют способ проверки результатов:

* **String** : Сравнение ожидаемого и актуального ответа как строк в консоли.
* **SaveActualData** : Режим «записи». Ассерты не выполняются, фреймворк просто собирает данные из кода для последующего
  сохранения.
* **File** : Режим интерактивного Diff. При несовпадении IDEA открывает стандартное окно сравнения файлов. Вы можете
  переносить данные из Actual в Expected визуально.
* **SingleFile** : Все упавшие тесты одного прогона объединяются в одно общее окно сравнения.

### Режим повторения (Repeat)

* **Once** : Однократный прогон.
* **UntilStopped** : Бесконечный цикл прогонов. Основной режим для Live Coding тестов.

## 2. Живое написание тестов (Live Test Development)

Режим UntilStopped в сочетании с **Debug** позволяет писать тесты «на лету» без перезапуска Spring-контекста.

### Технический механизм

В режиме UntilStopped внутренний итератор InfiniteTestIterator в конце каждого цикла вызывает метод init(), который *
*физически перечитывает XML-файл с диска** .

### Workflow разработки:

* **Точка остановки (Breakpoint)** : Поставьте брейкпоинт в классе DynamicTestBuilder.java в методе
  InfiniteTestIterator.init() на строке:

```
  Test test = testMapper.readAllTests();
```

* **Запуск** : Запустите тест в режиме **Debug** с настройками repeat = UntilStopped и assertion = File.
* **Итерация** :
  * Когда выполнение остановится на брейкпоинте, допишите в XML новый <test type="Case"> (достаточно только заголовка и
    запроса).
  * Нажмите **Resume (F9)** .
  * Фреймворк прочитает обновленный файл и сразу выполнит новый тест.
  * Если результат в открывшемся окне Diff верный, перенесите его в "Expected" и нажмите кнопку **Save file assertion**
    в плагине.
  * Повторяйте, не останавливая отладку.

## 3. Управление Моками (Mockito)

Плагин динамически меняет поведение объектов @TestBeanMock, @TestConstructorMock и @TestStaticMock:

* **useMocks** : Включает режим моков (environment=mock). Позволяет не запускать Docker-контейнеры локально.
* **mockAlwaysCallRealMethods** : Моки всегда вызывают реальные методы. Используется для автоматического сбора ответов
  из работающей системы.
* **mockCallRealMethodsOnNoData** : Если для вызова в XML нет секции <result>, будет выполнен реальный метод.
* **mockReturnMockOnNoData** : Если данных в XML нет, вернется пустой "Deep Mock".
* **spyCreateData** : Автоматически записывает результаты вызовов spy-объектов для сохранения в XML.

## 4. Отладка базы данных (Hibernate)

Позволяет включить детальные логи SQL без изменения application.properties:

* **hibernateShowSql / FormatSql** : Вывод и форматирование SQL в консоли.
* **hibernateUseSqlComments** : Добавление комментариев к запросам.
* **hibernateShowBindParams** : Включает TRACE для логгеров параметров (вы увидите значения вместо ?).

## 5. Кнопка «Save file assertion»

Выполняет физическую перезапись исходных XML-файлов тестов в папке src/test/resources.

**Принцип работы:**   
Плагин берет изменения, которые вы внесли во временные файлы в окне Diff, сопоставляет их с оригиналом на диске (через
служебные файлы settings.txt и template.xml) и патчит исходный код теста, сохраняя структуру XML и отступы.
