### Механизм ResettableIterator

В стандартной Java итераторы (`Iterator`) являются «одноразовыми». В интеграционных тестах это создает проблему: если
библиотека логирует объект (сериализует его) перед выполнением теста, итератор истощается, и основной код приложения
получает пустые данные.  
Механизм `ResettableIterator` в составе данной библиотеки решает эту проблему, обеспечивая **многократную сериализацию**
данных.

### 1. Ключевые особенности

* **Автоматическая обертка** : При десериализации (чтении JSON/XML) данных в тип `Iterator`, библиотека сохраняет
  элементы во внутренний список и возвращает реализацию `ResettableIterator`.
* **Авто-сброс (Reset)** : Сериализатор автоматически вызывает метод `.reset()` до и после записи данных в поток.
* **Сохранность данных**: Вы можете сериализовать один и тот же объект с итератором неограниченное число раз (например,
  для вывода в логи, передачи в MockMvc и использования в ассертах) --- данные всегда будут доступны с начала.

### 2. Пример использования

Если ваше DTO содержит итератор:  
java

    public class MyTestDto {
        private Iterator<String> data;
    }

При получении этого объекта через механизмы библиотеки, поле `data` будет содержать итератор, который не «сгорает» после
первого использования.
